## Overview

**TeGem** bridges your Telegram chats with Google Gemini by driving the Gemini web UI directly through [Playwright](https://playwright.dev/) browser automation. No Gemini API key, no billing — just a Google account and a Telegram bot token.

The bot is aware of its own commands and identity via a system prompt injected into Gemini at the start of each session. It streams responses back to Telegram in real-time by progressively editing the reply message as Gemini types, and it can capture and forward generated images.

---

## Features

| Feature | Details |
|---|---|
| **Real-time streaming** | Gemini's response is polled from the DOM and pushed to Telegram via message edits as text arrives |
| **Image generation** | `/imagine` captures images generated by Gemini and sends them as Telegram photos |
| **Command awareness** | System prompt tells Gemini its own identity and command list so it can explain itself naturally |
| **Session persistence** | Playwright profiles are saved to disk — no re-login needed after the first run |
| **Conversation context** | Stays in the same Gemini conversation across messages; `/clear` reloads a fresh session |
| **Quota detection** | Detects Gemini rate-limit messages and reports them gracefully to the user |
| **Graceful shutdown** | SIGINT/SIGTERM handled cleanly — closes browser and Telegram polling |
| **No official API** | Drives `gemini.google.com` directly via browser automation |

## Architecture

```
Telegram User
     │
     ▼
 grammY Bot (grammy)
     │
     ├── /start, /help, /clear, /status, /imagine
     │
     └── text messages
           │
           ▼
     GeminiProvider
       ├── sendPrompt()        →  types into rich-textarea, submits
       ├── streamResponse()    →  polls DOM, yields text deltas
       ├── captureImages()     →  extracts generated images as base64
       └── sanitize()          →  strips Gemini UI noise from text
           │
           ▼
     GeminiSessionManager (Playwright)
       └── Persistent BrowserContext (Chrome profile)
             │
             ▼
         gemini.google.com/app
```

## Requirements

- **Node.js** ≥ 20
- **Google Chrome** (or Chromium) installed locally
- A **Google account** logged into Gemini
- A **Telegram Bot Token** from [@BotFather](https://t.me/BotFather)

## Setup

### 1. Clone the repo

```bash
git clone git@github.com:0xfunboy/TeGem.git
cd TeGem
```

### 2. Install dependencies

```bash
npm install
```

### 3. Install Playwright browser (if not using system Chrome)

```bash
npm run playwright:install
```

### 4. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and set at minimum:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
```

Full options are documented in [.env.example](.env.example).

### 5. Run

```bash
# Development (hot-reload)
npm run dev

# Production build
npm run build
npm start
```

On first run, Chrome will open and navigate to `gemini.google.com`. Log in with your Google account. The session (cookies, localStorage) is saved to `.playwright/profiles/` and reused on subsequent starts — no further login needed.

## Bot Commands

| Command | Description |
|---|---|
| `/start` | Welcome message and introduction |
| `/help` | Show all available commands |
| `/clear` | Clear current conversation and start fresh |
| `/status` | Show bot and Gemini session status |
| `/imagine <description>` | Ask Gemini to generate an image |

Any other text message is forwarded to Gemini as a conversation turn. The first message of each session includes a system prompt that gives the bot its identity and command awareness.

## Configuration Reference

| Variable | Default | Description |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | — | **Required.** Token from @BotFather |
| `PLAYWRIGHT_HEADLESS` | `false` | Run Chrome without a visible window (requires Xvfb on servers) |
| `PLAYWRIGHT_EXECUTABLE_PATH` | auto | Path to Chrome binary |
| `PLAYWRIGHT_BROWSER_CHANNEL` | `chrome` | Browser channel: `chrome`, `msedge`, `chromium` |
| `GEMINI_PROFILE_DIR` | `_shared` | Profile directory name under `.playwright/profiles/<namespace>/` |
| `STREAM_POLL_INTERVAL_MS` | `700` | How often (ms) to read Gemini's DOM for new text |
| `STREAM_STABLE_TICKS` | `4` | Consecutive unchanged reads before declaring response complete |
| `STREAM_FIRST_CHUNK_TIMEOUT_MS` | `25000` | Max ms to wait for Gemini to start responding |
| `STREAM_MAX_DURATION_MS` | `90000` | Absolute max response duration |
| `SYSTEM_PROMPT` | built-in | Override the system prompt injected on session start |

## How Streaming Works

1. User sends a message → bot replies immediately with `"Sto elaborando…"`
2. Prompt is typed into Gemini's `rich-textarea` and submitted
3. A polling loop reads the `message-content` custom element every `STREAM_POLL_INTERVAL_MS` ms
4. Each new text delta is appended to the accumulated response and the Telegram message is edited with a `▌` cursor
5. After `STREAM_STABLE_TICKS` consecutive reads with no change, the loop finalizes
6. The message is edited one last time with the clean final text
7. If images were generated, they are extracted as base64 and sent as Telegram photos

## Headless Server Deployment

To run without a display (e.g. a VPS):

```bash
# Install Xvfb
sudo apt-get install -y xvfb

# Run with virtual display
Xvfb :99 -screen 0 1440x960x24 &
DISPLAY=:99 PLAYWRIGHT_HEADLESS=false npm start
```

Or set `PLAYWRIGHT_HEADLESS=true` if you have already saved a Gemini session profile locally and copied it to the server.
